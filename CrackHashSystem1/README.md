# CrackHashService

Актуальная версия распределенной системы перебора MD5-хэша на Spring Boot.

## Архитектура

- Пользователь <-> Менеджер: **JSON/HTTP**
- Менеджер <-> Воркеры: **XML/HTTP** (JAXB-модели по XSD)
- Хранение состояния запросов: потокобезопасные коллекции в памяти менеджера
- Запуск: `docker-compose` с 1 менеджером и 3 воркерами

## Сервисы

- `manager` (порт `8080`)
  - публичное API для пользователя
  - внутренний endpoint для ответов воркеров
  - веб-интерфейс проверки
- `worker1`, `worker2`, `worker3`
  - обработка диапазонов перебора
  - кооперативная отмена активных задач

## Запуск

```bash
docker compose up --build -d
```

Веб-интерфейс: [http://localhost:8080](http://localhost:8080)

## Публичное API менеджера (JSON)

### 1) Запуск взлома

`POST /api/hash/crack`

```json
{
  "hash": "e2fc714c4727ee9395f324cd2e7f331f",
  "maxLength": 4
}
```

Ответ:

```json
{
  "requestId": "730a04e6-4de9-41f9-9d5b-53b88b17afac"
}
```

### 2) Получение статуса

`GET /api/hash/status?requestId=<uuid>`

Пример:

```json
{
  "status": "IN_PROGRESS",
  "data": null
}
```

### 3) Отмена конкретной задачи

`POST /api/hash/cancel?requestId=<uuid>`

Возвращает текущий статус задачи.

### 4) Отмена всех активных задач

`POST /api/hash/cancel-all`

Пример ответа:

```json
{
  "canceledCount": 2,
  "requestIds": [
    "19471ef4-308b-4732-a7ce-642c49015941",
    "9ebc5f53-3ddb-4990-b7cb-54ecdd702d07"
  ]
}
```

### 5) Генерация MD5

`POST /api/hash/generate`

```json
{
  "word": "abcd"
}
```

Ответ:

```json
{
  "hash": "e2fc714c4727ee9395f324cd2e7f331f"
}
```

## Внутренние endpoint-ы (XML)

### Manager -> Worker

- `POST /internal/api/worker/hash/crack/task`
- `POST /internal/api/worker/hash/crack/cancel`

### Worker -> Manager

- `PATCH /internal/api/manager/hash/crack/request`

## Статусы запросов

- `IN_PROGRESS`
- `READY`
- `TIMEOUT`
- `CANCELED`
- `ERROR_NO_WORKERS`
- `ERROR_DISPATCH`
- `ERROR_WORKER_RESPONSE`
- `ERROR_NOT_FOUND`

## Бизнес-правила

- Алфавит перебора: `abcdefghijklmnopqrstuvwxyz0123456789`
- Пространство слов делится между воркерами по `partNumber/partCount`
- При `TIMEOUT` и любом `ERROR_*` менеджер автоматически отправляет отмену задачи на воркеры
- После отмены/ошибки/таймаута воркеры прекращают вычисления по задаче

## Веб-интерфейс

В интерфейсе доступны:

- генерация MD5
- запуск взлома
- просмотр статуса (автообновление каждую секунду)
- отмена конкретного запроса
- отмена всех активных запросов
- история задач с сохранением в `localStorage`
- очистка истории кнопкой

# Задание звучит следующим образом:
# Task 1. Services Implementation

Данная работа является первой частью проекта **CrackHash** — распределённой системы для взлома MD5-хэша.

Система выполняет подбор исходного слова по заданному хэшу методом полного перебора. Для генерации слов используется алфавит из строчных латинских букв и цифр.

В рамках первой лабораторной работы реализуются два сервиса:

- **Manager** — принимает запросы от клиента, разбивает задачу на части и отправляет их воркерам;
- **Worker** — получает часть задачи, перебирает слова в своём диапазоне и возвращает найденные результаты менеджеру.

Взаимодействие между сервисами организуется по протоколу **HTTP**.

---

## Общая логика работы системы

1. Пользователь отправляет менеджеру MD5-хэш некоторого слова и максимальную длину слова.

2. Менеджер принимает запрос и создаёт уникальный идентификатор задачи.

3. Менеджер делит общее пространство перебора на части в соответствии с количеством воркеров.

4. Каждому воркеру отправляется отдельная задача на перебор слов в заданном диапазоне.

5. Воркер перебирает слова, вычисляет их MD5-хэш и сравнивает его с переданным значением.

6. Если совпадения найдены, воркер возвращает результат менеджеру.

7. После получения ответов от всех воркеров менеджер переводит задачу в статус `READY`.

8. Клиент может получить результат по идентификатору запроса.

---

## Цель лабораторной работы

Цель работы — реализовать базовую версию распределённой системы CrackHash и организовать простое взаимодействие между менеджером и воркером через HTTP.

В рамках данной задачи не требуется настройка очередей, producer-ов и listener-ов.
